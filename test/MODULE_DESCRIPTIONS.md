# Hotel Management System - Module Descriptions

## QUAN'S MODULES

### MODULE 1: BOOKING MODULE (TC_BM_01 to TC_BM_20)
**Description:** Manage room inventory including add, update, delete rooms with validation for room number, type, bed, price, and status.

### MODULE 2: CUSTOMER MODULE (TC_CM_01 to TC_CM_20)
**Description:** Handle customer check-in and check-out processes with customer information validation and room assignment.

### MODULE 3: NOTIFICATION SYSTEM (TC_NS_01 to TC_NS_10)
**Description:** Display user messages including success messages, error messages, confirmation dialogs, and informational notifications.

### MODULE 4: WEBSITE HOME/DASHBOARD (TC_DH_01 to TC_DH_10)
**Description:** Provide main navigation interface with dashboard buttons and links to access different modules.

### MODULE 5: ADMIN MANAGEMENT (TC_AM_01 to TC_AM_10)
**Description:** Manage user accounts by searching users, viewing user information, and changing user status between approved and pending.

---

## LAM'S MODULES

### MODULE 1: LOGIN/LOGOUT (TC_LL_01 to TC_LL_15)
**Description:** Handle user authentication for admin and regular users with credential validation and session management.

### MODULE 2: ROOM MANAGEMENT (TC_RM_01 to TC_RM_15)
**Description:** Manage room inventory operations including add, update, delete rooms with complete validation and data integrity checks.

### MODULE 3: PAYMENT AND INVOICE (TC_PI_01 to TC_PI_15)
**Description:** Calculate payment amounts based on room price and days stayed, generate invoices, and store bill information.

### MODULE 4: DASHBOARD LAYOUT (TC_DL_01 to TC_DL_10)
**Description:** Manage visual presentation and layout of the main dashboard interface with UI elements and navigation.

### MODULE 5: REPORT GENERATION (TC_RG_01 to TC_RG_10)
**Description:** Retrieve and display previously generated invoices and bills by searching with bill ID.

### MODULE 6: DATABASE INTEGRITY (TC_DI_01 to TC_DI_05)
**Description:** Ensure data consistency and integrity across the system with foreign key validation and constraint checks.
